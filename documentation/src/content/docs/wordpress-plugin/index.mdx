Loopress ships as a single WordPress plugin, built in two editions from the same codebase:

- **Loopress** (free, wordpress.org): connects your site to the Loopress CLI. It integrates with [Code Snippets](https://wordpress.org/plugins/code-snippets/) and [WPCode](https://wordpress.org/plugins/wpcode/) and exposes the REST API that powers `lps snippet pull/push/list`.
- **Loopress Plus** (also free, loopress.dev only): everything in the free edition, plus Composer dependency management, search Packagist, install and audit PHP packages from the admin, and sync `composer.json`/`composer.lock` with `lps composer push/pull`.

Loopress Plus costs nothing, it is not a paid upgrade. It is not distributed through the wordpress.org directory because directory guidelines do not allow a plugin whose purpose is installing executable PHP code from external registries such as Packagist, not because of pricing. Installing Loopress Plus deactivates the free edition automatically, since Plus is a full replacement, not an add-on; the two editions never run side by side.

## Loopress

<PluginDownload edition="free" />

- **Snippet sync**: pull the site's snippets as files with `lps snippet pull`, push edits back with `lps snippet push`, and keep them in Git
- **Works with Code Snippets and WPCode**: Loopress talks to whichever is active, no migration needed
- **REST API for the CLI**: everything runs over `loopress/v1`, restricted to the `manage_options` capability (administrator role). `lps plugin pull/push/add` uses WordPress core's own `wp/v2/plugins` API instead.

## Loopress Plus

<PluginDownload edition="plus" />

Everything above, plus:

- **Dependency management**: search, install, remove and update Composer packages from [Packagist](https://packagist.org/), with automatic detection of outdated packages
- **Security audit**: surfaces known CVEs and abandoned packages among your installed dependencies
- **Platform diagnostics**: catches PHP version mismatches before they cause a broken install
- **CLI sync**: powers `lps composer push` and `lps composer pull`

Loopress Plus stores your project's Composer dependencies in `wp-content/loopress/`, separate from the plugin itself, so they survive plugin updates.

## About wordpress.org-hosted plugins

Plugins listed on the [wordpress.org Plugin Directory](https://wordpress.org/plugins/) go through a review by the WordPress.org plugins team before their first release, and every update is scanned afterward. In exchange, a hosted plugin gets:

- **Discoverability**: found by searching **Plugins → Add New** from inside any WordPress admin, no external download needed
- **Automatic updates**: WordPress checks wordpress.org for new versions and can update the plugin in place, the same as any other installed plugin
- **A trust signal**: install counts, ratings, and a public source history are visible to anyone considering the plugin

The tradeoff is a strict set of guidelines. One of them, Rule 8 of the Plugin Directory guidelines, forbids a plugin from sending or installing executable code from a third-party system, which includes package registries such as Packagist. That is exactly what Loopress Plus's Composer dependency management does (searching Packagist and installing PHP packages), so it cannot be hosted on wordpress.org regardless of implementation quality. The free Loopress edition has no such capability, so it goes through the standard wordpress.org submission and update process.

This is why Loopress ships as two separate downloads instead of one wordpress.org listing: the free edition follows the usual wordpress.org install and update flow, while Loopress Plus is downloaded directly from loopress.dev and updated manually by re-uploading a new `loopress-plus.zip` when one is released.

## Requirements

- WordPress 6.0+
- PHP 8.2+
- `manage_options` capability (administrator role)

## Installation

1. Download `loopress.zip` (free) or `loopress-plus.zip` using the buttons above, depending on whether you need Composer dependency management
2. In your WordPress admin, go to **Plugins → Add New → Upload Plugin**
3. Upload and activate the plugin, then find **Loopress** in the sidebar
